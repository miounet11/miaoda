## Overview
Implement a comprehensive conversation organization system that allows users to categorize, tag, and manage their chats effectively. This includes folder/tag systems, bulk operations, conversation categorization, and organizational tools that scale with large chat histories.

## Problem Statement
Users with extensive chat histories need better organizational tools to manage and categorize their conversations. Without proper organization features, finding specific types of conversations becomes difficult and workspace management becomes chaotic.

## Acceptance Criteria
- [ ] Implement tag-based conversation organization system
- [ ] Add folder/category hierarchy for chat grouping
- [ ] Provide bulk operations for conversation management
- [ ] Implement conversation starring and pinning functionality
- [ ] Add conversation archiving and hiding capabilities
- [ ] Create smart folders based on conversation metadata
- [ ] Implement drag-and-drop organization interface
- [ ] Add conversation templates for common use cases
- [ ] Support import/export of organizational data
- [ ] Ensure organization persists across application restarts

## Technical Details

### Organization Features
- **Tags System**: User-defined tags with color coding and filtering
- **Folders**: Hierarchical folder structure for conversation grouping
- **Smart Folders**: Auto-generated folders based on rules (date, participants, etc.)
- **Bulk Operations**: Multi-select for tagging, moving, archiving
- **Conversation Metadata**: Enhanced metadata for better organization

### Database Schema Changes
- New tables for tags, folders, and conversation relationships
- Migration scripts for existing conversations
- Indexes for efficient organization queries
- Support for many-to-many tag relationships

### UI Components
- Sidebar organization panel with expandable folders
- Tag management interface with CRUD operations
- Bulk action toolbar for selected conversations
- Drag-and-drop interface for easy organization
- Organization settings and preferences

### Files to Modify/Create
- `src/main/db/organization.ts` - Organization database layer
- `src/renderer/src/components/organization/` - Organization UI components
- `src/renderer/src/stores/organization.ts` - Organization state management
- `src/renderer/src/services/OrganizationService.ts` - Business logic
- `src/main/db/migrations/` - Schema migrations for organization tables
