## Overview
Optimize the SQLite database performance for MiaoDa Chat to handle large conversation histories efficiently. This foundational task focuses on indexing strategies, query optimization, and performance profiling to ensure the application remains responsive as data volume grows.

## Problem Statement
As users accumulate large chat histories, the application may experience performance degradation in:
- Message loading and pagination
- Search functionality
- Database startup times
- Memory usage during large queries

## Acceptance Criteria
- [ ] Implement optimized database indexes for common query patterns
- [ ] Profile and optimize existing SQL queries in the codebase
- [ ] Establish performance benchmarks for database operations
- [ ] Implement query result caching where appropriate
- [ ] Add database maintenance routines (VACUUM, ANALYZE)
- [ ] Document database performance best practices
- [ ] Ensure database operations remain under 100ms for typical queries
- [ ] Support databases with 100,000+ messages without performance degradation

## Technical Details

### Database Schema Optimization
- Analyze current schema in `src/main/db/` directory
- Add compound indexes for message filtering (chat_id + timestamp)
- Optimize full-text search indexes for `search_index` table
- Consider partitioning strategies for large message tables

### Query Optimization Areas
- Message pagination queries in database layer
- Search functionality performance
- Chat list loading with message previews
- Attachment handling and metadata queries

### Performance Monitoring
- Add query timing instrumentation
- Implement database health checks
- Create performance regression tests
- Monitor memory usage patterns

### Files to Modify
- `src/main/db/database.ts` - Core database operations
- `src/main/db/migrations/` - New migration files for indexes
- `src/main/ipcHandlers.ts` - Query optimization
- `src/test/` - Performance test suite
